Bug

2019-11-24

Bug的管理：
管理平台：禅道、Bugzilla、Bug Free、ALM、QC、Test link
六要素：编号，bug名称，bug优先级（高中低），bug严重级（致命——与钱有关，崩溃闪退，影响产品核心流程正常使用；严重；一般；轻微——用户体验UI），bug复现步骤，附件（截图，日志，video），复制用例步骤，预期结果，实际结果
生命周期：从发现bug到解决bug
状态
等级
对Bug的统计分析从哪些方面着手：解决时间，每个版本，状态，测试，开发，软件模块，等级对被测软件的质量评估（不同公司的要求不同！）：1、2级bug全关闭了，3级bug关闭80%，4级bug无所谓。
Bug的状态：新建new，打开、激活open，已确认，已解决，拒绝，重新打开，关闭，延期处理重复bug
